# Deduplication Strategy (중복 제거 전략)

**대상 독자**: RAG 검색 품질 엔지니어
**목적**: 무의미하게 중복되는 청크(Chunks)가 벡터 데이터베이스에 삽입되는 현상을 통제하는 Near-Duplicate 로직을 숙지합니다.
**범위**: `ragprep/core/dedupe.py` 의 N-gram 및 Jaccard 인메모리 필터 로직. 

---

## 1. Near Duplicate의 개념과 도입 배경

만약 문서들의 서론 파트나 법적 고지(Disclaimer)와 같이 수많은 파일 간에 겹치는 본문이 있다면 어떻게 될까요? RAG 파이프라인에서 중복은 단순히 인덱스 용량을 차지하는 것을 넘어, Vector Search 검색 시 상위 Top-K 응답을 모두 동일한 "오염된" 청크가 덮어버리는 치명적 결과(Context Pollution)를 초래합니다.

따라서 단순히 문단이 100% 똑같은 일치(Exact Match) 여부를 넘어, 비슷한 단어가 다수 포진된 Near-Duplicate 텍스트를 발견해 폐기해야 합니다.

## 2. N-gram 핑거프린트 엔진 구조

본 파이프라인의 `dedupe.py` 모듈은 SimHash 등의 외부 의존성 없이 순수 인메모리 방식으로 지문(Fingerprint)를 추출합니다.

```mermaid
graph TD
    A[입력 텍스트 청크] -->|소문자 변환| B[단어 분리 Tokenize]
    B -->|N=3 묶음| C[N-gram 집합 생성]
    C --> D[기존 핑거프린트 DB (Memory)]
    D --> E{Jaccard 유사도 계산}
    
    E -- "유사도 >= 0.85" --> F(중복 판정 - SKIP)
    E -- "유사도 < 0.85" --> G(신규 청크 판정 - SAVE)
    G --> H[핑거프린트 DB에 등록]
```

- **Jaccard Similarity**: N-gram 단어 집합 간의 `(교집합 크기) / (합집합 크기)` 비율을 측정하며, 기본 임계치인 `0.85`(텍스트의 85% 이상이 일치)를 넘으면 통계적으로 중복으로 판명합니다.
- **오버헤드 방지**: 아주 짧은 텍스트나 빈 텍스트는 자동으로 우회(Bypass)합니다.

## 3. Dedupe Scope (중복 스코프)

`--dedupe-scope` 인자를 통해 중복 필터가 작동하는 "기억의 한계"를 지정합니다. 이 범위 지정으로 시스템 복잡도와 RAG 정책을 조율할 수 있습니다.

| Option | 동작 방식 설명 (Explanation) | 장점 및 권장 (Pros) |
| :--- | :--- | :--- |
| `doc` | **단일 문서 내 배제**: 각 문서(doc_id) 안에서 방금 전 문단이나 이전 장(Chapter)에 나타난 텍스트와 겹칠 때만 중복을 던집니다. 다른 문서가 불릴 때는 지문 DB가 리셋됩니다. | 리소스 사용량이 가장 적으며 동일 문서 내 헤더 오작동이나 꼬리말 중복 제거에 효율적. (기본값) |
| `group` | **그룹 전체 배제**: `--merge-group` 환경이나 같은 `group_id` 폴더 하층에 있는 수십 개의 문서 풀 간에 겹치는 부분까지 배제합니다. 메모리를 조금 더 먹습니다. | 방대한 성경 파일이나 사내 규정집처럼 서로 다른 파일에 연속되는 내용이 퍼져있을 때 매우 강력함. |
